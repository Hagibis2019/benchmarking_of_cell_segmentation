# Restoring Results

## 9.1 Undo and Redo

Click the <kbd>Undo</kbd> button in the <kbd>Interaction</kbd> region, or press the shortcut <kbd>Shift</kbd>+<kbd>Z</kbd>, to undo the previous operation and restore the label data to its state before modification.

Click the <kbd>Redo</kbd> button in the <kbd>Interaction</kbd> region, or press the shortcut <kbd>Shift</kbd>+<kbd>B</kbd>, to restore the operation that was just undone.

NuPatch3D supports up to 10 steps of undo and redo. For napari brush, eraser, fill, and other operations, use <kbd>Ctrl</kbd>+<kbd>Z</kbd> to undo.


## 9.2 Restoring Results Using JSON Logs

When the program exits abnormally, label data is accidentally lost, or previous refinement results need to be recovered, the original image, initial pre-segmented label image, and the corresponding JSON log file can be used to reconstruct the refined results.

Click <kbd>Apply JSON Log</kbd> and select the JSON log file generated by NuPatch3D. NuPatch3D will automatically read the historical records in the log and re-apply the committed modifications to the current label layer in the saved order.

During the recovery process, NuPatch3D will automatically check the compatibility between the log file and the current layer. If the layer names recorded in the log are inconsistent with the layer names in the current napari, the system will try to automatically match the appropriate layer; if it cannot uniquely determine the target layer, a dialog box will pop up for manual selection.

After recovery is completed, the label data will be restored to the latest commit state recorded in the log file, and you can continue subsequent editing or directly export the results.

> **Tip**
>
> To ensure correct recovery results, it is recommended to use the original image and initial pre-segmented label image that correspond to the JSON log file when it was generated.
